# Промпт для создания MealGen в lovable.dev

Создай веб-приложение для генерации планов питания и списков покупок.

## Описание проекта

MealGen - это веб-приложение для создания персонализированных планов питания с использованием AI. Пользователи могут указать доступные ингредиенты, получить план питания на неделю и автоматически сгенерировать список покупок.

## Основные функции

1. Генерация плана питания:
   - 3 приема пищи в день (завтрак, обед, ужин)
   - Разделение на будни (4 дня) и выходные (3 дня)
   - Учет доступных ингредиентов
   - Расчет на 2 человека по умолчанию

2. Управление рецептами:
   - Просмотр деталей рецепта
   - Регенерация отдельных блюд
   - Редактирование ингредиентов

3. Список покупок:
   - Автоматическая генерация
   - Консолидация ингредиентов
   - Копирование в буфер обмена

## Интерфейс

### Главная страница (index.html)

Разделена на 3 колонки:

1. Левая колонка (250px):
   - Заголовок "Пожелания"
   - Кнопка "Создать меню"
   - Поле ввода ингредиентов
   - Список доступных ингредиентов

2. Центральная колонка:
   - Заголовок "Меню"
   - Сетка с рецептами:
     * Строки: завтрак, обед, ужин
     * Колонки: будни, выходные
   - Для каждого рецепта:
     * Название блюда
     * Кнопка просмотра рецепта
     * Кнопка регенерации

3. Правая колонка (250px):
   - Заголовок "Список"
   - Кнопка копирования
   - Список покупок

### Модальное окно рецепта

Содержит:
- Заголовок с названием блюда
- Кнопки редактирования и закрытия
- Список ингредиентов с количествами
- Пошаговые инструкции

## API Endpoints

### POST /generate
Генерирует план питания
```typescript
interface GenerateRequest {
    availableIngredients: string[];
}

interface GenerateResponse {
    meal_plan: Meal[];
}
```

### POST /regenerate_meal
Регенерирует отдельное блюдо
```typescript
interface RegenerateMealRequest {
    excludedIngredients?: string[];
    availableIngredients?: string[];
    requiredIngredients?: string[];
    mealIndex: string;
    mealType: string;
}

interface RegenerateMealResponse {
    meal: Meal;
}
```

### POST /generate_shopping_list
Создает список покупок
```typescript
interface ShoppingListRequest {
    meals: Meal[];
}

interface ShoppingListResponse {
    shopping_list: string[];
}
```

## Модели данных

### Meal
```typescript
interface Meal {
    name: string;
    ingredients: string[];
    instructions: string[];
    cooking_time: number;
    calories_per_serving: number;
    meal_type?: string;
}
```

### Ingredient
```typescript
interface Ingredient {
    name: string;
    amount: number;
    unit: string;
}
```

### ShoppingList
```typescript
interface ShoppingList {
    items: { [key: string]: ShoppingItem };
}
```

## Требования к функциональности

1. Сохранение состояния:
   - Использовать LocalStorage для:
     * Текущего плана питания
     * Списка доступных ингредиентов

2. Валидация:
   - Проверка уникальности блюд
   - Валидация ингредиентов
   - Проверка форматов данных

3. Обработка ошибок:
   - Кастомные страницы 404 и 500
   - Информативные сообщения об ошибках
   - Возможность повторных попыток

4. Интерактивность:
   - Анимации при загрузке
   - Мгновенная обратная связь
   - Плавные переходы

## Дополнительные требования

1. Диетические рекомендации:
   - 2000 ккал в день
   - Соотношение БЖУ: 20/30/50
   - Без повторения основных ингредиентов

2. Генерация рецептов:
   - Разнообразие кухонь мира
   - Различные техники приготовления
   - Учет сезонности продуктов

3. Безопасность:
   - Защита от XSS
   - Валидация входных данных
   - Безопасная работа с API
